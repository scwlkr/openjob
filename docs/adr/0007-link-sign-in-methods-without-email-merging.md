# Link Sign-in Methods without email merging

OpenJob offers Google and Apple as Sign-in Methods on web, iOS, and Android, and lets one User link both while preserving the same User ID, immutable Username, Groups, and Tasks. Linking requires an authenticated OpenJob session, fresh authentication with the additional provider, and explicit confirmation; OpenJob never links or merges Users merely because provider email addresses match. This adds deliberate linking friction to prevent account takeover and incorrect merges while allowing one OpenJob identity to move cleanly between platforms.
