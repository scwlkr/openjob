# Link Sign-in Methods without email merging

OpenJob offers Google and Apple as Sign-in Methods on web, iOS, and Android, and lets one User link both while preserving the same User ID, immutable Username, Groups, and Tasks. Linking requires an authenticated OpenJob session, fresh authentication with the additional provider, and explicit confirmation; OpenJob never links or merges Users merely because provider email addresses match. When a successful provider credential is not linked, OpenJob explicitly asks whether to create a new User or link an existing User instead of silently creating another identity. If both methods already belong to different Users, OpenJob may remove an authenticated empty onboarding shell with no Username, Groups, or history before linking, but it does not automatically merge two Users with domain history. This adds deliberate linking friction to prevent account takeover and incorrect or accidental duplicate Users while allowing one OpenJob identity to move cleanly between platforms.
