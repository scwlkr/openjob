# Stage and sign native over-the-air updates

OpenJob currently ships every preview and production change through App Store and Play Store builds; development builds remain internal. Its Expo organization remains on Free, where signed EAS Update delivery is unavailable, so OTA is disabled rather than weakening update authenticity. Enabling Expo OTA later requires a separate owner-approved decision and paid signed-delivery entitlement; every production update would then be signed, verified on an internal channel on both iOS and Android, introduced through a gradual rollout, and kept rollback-ready. Any future OTA implementation must retain a working embedded bundle, download only compatible signed updates without blocking launch, and apply them on the next cold launch rather than restarting an active session.

Routine releases never block app use. OpenJob may remotely enforce a minimum store build only when a serious security issue or a genuinely incompatible API makes continued use unsafe. An unsupported build presents one clear explanation and a direct App Store or Play Store action rather than failing ordinary requests mysteriously; all other available updates remain optional and may be suggested quietly.
