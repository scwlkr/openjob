# Use best-effort standards-based Web Push

OpenJob uses standards-based Web Push so supported browsers, including installed iPhone and iPad Home Screen apps, share one feature-detected delivery path. Delivery runs asynchronously only after a successful Task change; a failed or missed notification never fails the Task action, and v1 has no durable retry pipeline. Triggers belong to the hosted API, so web and CLI actions behave identically. The service worker presents an OS notification even while OpenJob is visible and uses a Task-scoped identity so a newer event replaces the older visible notification for that Task. Each browser subscription persists through sign-out while its OpenJob User association pauses, then resumes only when the same User returns; another User must explicitly enable notifications for themselves. OpenJob offers enablement once at the first usable Group, does not repeat that prompt after dismissal or denial, and keeps a permanent Notifications control; disabling pauses server delivery without deleting the browser subscription.
